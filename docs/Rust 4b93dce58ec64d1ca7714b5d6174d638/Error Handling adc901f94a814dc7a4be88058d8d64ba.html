대부분의 언어는 에러 처리를 Exception으로 퉁치지만 Rust는 2가지 종류의 에러가 있다.

  • Recoverable Error such as a file not found error, it’s reasonable to report the problem to the user and retry the operation. Result<T, E>
  • Unrecoverable Error Unrecoverable errors are always symptoms of bugs, like trying to access a location beyond the end of an array. panic! macro : Stop execution when the program encounters unrecoverable error

Unrecoverable Errors with panic!

Unwinding the Stack or Aborting in Response to a Panic

원래는 패닉이 발생하면 Rust는 스택을 역순으로 순회하면서 각 함수에 전달되었던 데이터를 정리(unwind). 하지만 이 작업 양이 매우 클 수 있음. 따라서 스택을 정리하지 않고 즉시 종료(abort)하게 하려면 아래 옵션을 Cargo.toml에 추가

[profile.release]
panic = 'abort'

Simple panic

fn main() {
    panic!("crash and burn");
}

Using a panic! Backtrace

유효하지 않은 인덱스를 이용해 벡터에 접근하려고 함. Rust는 이 때 panic!을 발생시키고 실행중단시킴. C같은 언어는 원치 않던 값이라도 개발자가 지정한 위치의 메모리에 저장된 값을 리턴. 이것을 buffer overread라 하는데 보안 취약점이 될 수 있음.

환경변수 RUST_BACKTRACE=1 설정하면 backtrace 가능

fn main() {
    let v = vec![1, 2, 3];

    v[99];
}

Recoverable Errors with Result

file not found error 같은 건 프로그램을 완전히 종료하지 않고 새 파일을 생성해줌으로써 해결할 수 있음. 이런 대처가능한 에러들을 Result<T, E>로 처리

use std::fs::File;
use std::io::ErrorKind;

fn main() {
    let f = File::open("hello.txt");

    let f = match f {
        Ok(file) => file,
        Err(error) => match error.kind() {
            ErrorKind::NotFound => match File::create("hello.txt") {
                Ok(fc) => fc,
                Err(e) => panic!("Problem creating the file: {:?}", e),
            },
            other_error => {
                panic!("Problem opening the file: {:?}", other_error)
            }
        },
    };
}
